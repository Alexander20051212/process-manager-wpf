Process Manager (WPF + MVVM)

Настольное приложение для управления процессами операционной системы Windows, разработанное на платформе WPF с использованием архитектурного шаблона MVVM.

📌 Описание проекта

Данное приложение предназначено для получения, анализа и управления активными процессами операционной системы Windows.

Программа позволяет:

Просматривать список запущенных процессов

Просматривать потоки выбранного процесса

Изменять приоритет процесса

Завершать процессы

Управлять CPU Affinity (закрепление за логическими процессорами)

Отслеживать загрузку CPU и RAM

Выполнять поиск процессов по имени

Отображать дерево процессов

Обновление данных выполняется автоматически каждые 7 секунд.

⚙ Реализация CPU Affinity

CPU Affinity реализована через свойство:

Process.ProcessorAffinity

Для закрепления процесса за выбранными логическими процессорами используется битовая маска:

каждый бит соответствует логическому процессору

значение 1 — разрешено выполнение

значение 0 — выполнение запрещено

Количество логических процессоров определяется через:

Environment.ProcessorCount

📊 Мониторинг системы

Для получения информации о загрузке CPU и RAM используются системные счётчики производительности (PerformanceCounter):

Processor - % Processor Time

Memory - Available MBytes

Обновление показателей выполняется с использованием DispatcherTimer.

🔎 Принцип работы приложения

ProcessService получает данные о процессах через System.Diagnostics.Process.

Данные преобразуются в модели (ProcessInfo, ThreadInfo).

MainViewModel управляет логикой и командами.

View (XAML) обновляется через механизм Data Binding.

Обновление выполняется периодически без блокировки пользовательского интерфейса.
